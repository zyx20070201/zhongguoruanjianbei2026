# 10. 系统功能说明（界面截图及说明）

本节展示“大学生智能学习辅助平台”的主要系统界面。截图来自本地运行的软件原型，演示数据使用“数据库系统课程学习空间”，用于说明系统已经具备课程空间管理、学习资料管理、学习现场管理、知识图谱、学习计划、学习画像和 AI Terminal 等基本功能。

## 10.1 用户登录界面

![用户登录界面](screenshots/01-login.png)

图 10-1 为系统登录界面。用户可通过用户名或邮箱登录系统，也可以切换到注册入口创建账号。登录成功后，系统会建立用户会话，并进入课程空间页面。该功能对应数据库中的 `Users` 表，用户登录后创建的课程空间、学习资料、学习现场和学习事件都与当前用户关联。

## 10.2 课程空间与学习现场列表

![课程空间与学习现场列表](screenshots/02-workspace-home.png)

图 10-2 展示课程空间主页。左侧为系统导航区，包含新建学习现场、新建课程空间、搜索、课程空间列表和学习现场列表；右侧为当前课程空间的学习现场视图。示例中课程空间为“数据库系统课程学习空间”，其中包含“SQL JOIN 与知识图谱复习”学习现场。

该界面对应数据库中的 `Workspaces`、`Workbenches` 表。一个用户可以创建多个课程空间，一个课程空间可以包含多个学习现场。

## 10.3 学习现场管理界面

![学习现场管理界面](screenshots/03-workspace-workbenches.png)

图 10-3 展示课程空间中的学习现场管理功能。用户可以查看已有学习现场，并通过右上角按钮创建新的学习现场。学习现场用于组织一个具体学习任务，例如 SQL 多表连接复习、范式整理、事务练习等。

该界面对应数据库中的 `Workbenches` 表，也与 `WorkbenchResources` 表相关。学习现场创建后，可以进一步绑定学习资料、AI 对话、学习计划和学习事件。

## 10.4 课程资料管理界面

![课程资料管理界面](screenshots/04-knowledge-resources.png)

图 10-4 展示课程资料管理界面。系统将学习资料统一放在 Knowledge 标签页下管理，支持搜索资料、筛选资料类型、查看资料名称、类型、来源和更新时间。示例中包含“数据库范式与事务资料.md”和“SQL 连接查询复习笔记.md”两份资料。

该功能对应数据库中的 `FileObjects` 表。学习资料可以是课程文档、复习笔记、网页资料、AI 生成内容等，也可以被绑定到不同学习现场中复用。

## 10.5 课程知识图谱界面

![课程知识图谱界面](screenshots/05-knowledge-graph.png)

图 10-5 展示课程知识图谱界面。页面顶部显示当前图谱包含 4 个知识概念和 3 条概念关系。右侧为图谱分析区域，可用于查看目标结构、差距分析和概念详情。示例图谱包含“关系模型”“SQL 连接查询”“数据库范式”“事务 ACID”等概念。

该功能对应数据库中的 `KnowledgeConcepts`、`KnowledgeRelations`、`KnowledgeBindings` 表。知识概念之间可以建立先修、相关、包含等关系，从而支持学习路径分析和薄弱点定位。

## 10.6 学习现场工作台界面

![学习现场工作台界面](screenshots/10-workbench-add-resource.png)

图 10-6 展示学习现场工作台。用户进入某个学习现场后，可以打开课程资料、笔记或生成资源作为工作台标签页。截图中点击“Add first tab”后，系统显示可加入当前学习现场的资料列表，包括 SQL 复习笔记和范式事务资料。

该功能对应数据库中的 `Workbenches`、`FileObjects` 和 `WorkbenchResources` 表。`WorkbenchResources` 表用于保存学习现场与资料之间的多对多绑定关系。

## 10.7 AI Terminal 界面

![AI Terminal 界面](screenshots/07-ai-terminal.png)

图 10-7 展示课程级 AI Terminal。用户可以在该界面中输入学习目标、提问课程资料内容、请求生成学习计划，或让系统基于当前课程资料进行总结和解释。AI Terminal 是课程空间中的智能交互入口。

该功能对应数据库中的 `ConversationSessions`、`ConversationMessages` 和 `LearningEvents` 表。系统可以保存 AI 对话会话、每轮消息内容，并将关键交互记录为学习事件。

## 10.8 学习计划管理界面

![学习计划管理界面](screenshots/08-planning.png)

图 10-8 展示学习计划管理界面。用户可以在 Planning 标签页中新建学习计划，围绕学习目标生成阶段任务、复习安排和后续行动。截图中展示了学习计划入口和新建计划按钮。

该功能对应数据库中的 `LearningGoals` 和 `LearningPlans` 表。学习目标用于描述用户想完成的学习任务，学习计划用于保存围绕目标生成的阶段步骤和计划版本。

## 10.9 学习画像与记忆治理界面

![学习画像与记忆治理界面](screenshots/09-profile.png)

图 10-9 展示学习画像界面。页面上方为 Learner Timeline，用于展示学习行为和画像信号变化；下方包含 Profile Ledger 和 Memory Governance 区域。示例中可以看到近期事件 `flashcard.deck_created`，说明系统已经记录了闪卡卡组创建行为。

该功能对应数据库中的 `LearningEvents`、`SavedMemory`、学习画像相关表以及闪卡相关表。学习事件可作为后续学习画像、学习建议和复习策略的证据来源。

## 10.10 功能与数据库表对应关系

| 系统功能 | 界面截图 | 主要数据库表 |
| --- | --- | --- |
| 用户登录与账号管理 | 图 10-1 | `Users` |
| 课程空间管理 | 图 10-2 | `Workspaces` |
| 学习现场管理 | 图 10-3 | `Workbenches` |
| 课程资料管理 | 图 10-4 | `FileObjects` |
| 课程知识图谱 | 图 10-5 | `KnowledgeConcepts`、`KnowledgeRelations`、`KnowledgeBindings` |
| 学习现场资源绑定 | 图 10-6 | `WorkbenchResources` |
| AI 对话与学习事件 | 图 10-7 | `ConversationSessions`、`ConversationMessages`、`LearningEvents` |
| 学习目标与计划 | 图 10-8 | `LearningGoals`、`LearningPlans` |
| 学习画像与记忆治理 | 图 10-9 | `LearningEvents`、`SavedMemory` |

通过以上界面可以看出，系统已经具备数据库实验要求中的基本增删改查原型基础：用户可以创建课程空间、管理学习资料、创建学习现场、查看知识图谱、进入 AI Terminal，并通过学习事件和学习画像界面查看学习过程数据。
