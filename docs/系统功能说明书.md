# AI Learning Workspace 系统功能说明书

版本：V0.1  
状态：功能框架初稿  
适用对象：参赛评审、指导教师、项目开发与测试人员、系统演示人员  

## 1. 文档说明

本文档用于说明 AI Learning Workspace 的用户使用方式与系统功能框架。当前版本优先搭建完整的软件功能结构，后续可继续补充详细操作步骤、界面截图、典型案例、测试数据与演示流程。

AI Learning Workspace 是一个面向新时代大学生自主学习、课程资料管理、知识建模与个性化学习规划的 AI 原生学习工作空间。系统围绕“课程 Workspace”和“学习现场 Workbench”组织学习过程，将文件资料、AI 对话、知识图谱、学习诊断、学习计划、资源生成、记忆管理和多模态资料解析整合在同一平台中。

## 2. 系统定位

### 2.1 产品定位

系统定位为面向高校学生的智能学习工作台，帮助学生在课程学习、考试复习、项目实践、文献阅读和知识整理等场景中完成以下任务：

- 管理课程资料、笔记、代码、网页来源、音视频资料和 AI 生成内容。
- 基于用户目标创建独立的学习现场，形成围绕任务的资源、笔记、代码、练习和 AI 辅助空间。
- 将课程资料抽取为知识块和知识图谱，支持概念关系、先修依赖、薄弱点定位和补救路径分析。
- 通过 AI Terminal、多智能体编排和学习规划能力，为学生生成学习目标、阶段计划、练习资源和复盘建议。
- 基于学习事件、对话历史、显式记忆和掌握度信号，维护可解释的学习画像。
- 通过 AI Studio 生成测验、闪卡、思维导图、报告、表格、可视化讲解等学习资源。

### 2.2 目标用户

- 大学生：用于课程学习、资料整理、复习备考、项目实践和自主学习规划。
- 教师或助教：用于演示智能学习空间、组织课程资料、观察学习状态。
- 竞赛评审：用于快速理解系统的功能覆盖、AI 融合方式和用户体验设计。
- 开发与测试人员：用于理解系统模块边界，编写测试用例和演示脚本。

### 2.3 典型使用场景

- 课程学习：用户创建“数据结构”“数据库系统”“机器学习”等课程空间，上传课件、教材、作业和代码。
- 考试复习：用户输入复习目标，系统生成阶段计划、诊断题、薄弱知识点与复习卡片。
- 项目实践：用户为一个项目任务创建 Workbench，集中管理需求、代码、笔记、AI 讨论和生成资源。
- 文献阅读：用户导入网页、PDF 或文本资料，系统提取内容、建立索引，并支持基于来源的问答。
- 知识补弱：系统根据知识图谱和学习画像识别薄弱概念，给出先修补救路径。
- 多模态学习：用户上传音频或视频资料，系统提取转写、摘要、关键片段和资源智能分析结果。

## 3. 用户角色与权限

### 3.1 普通学习用户

普通学习用户是系统的主要使用者，可完成注册登录、创建课程空间、管理资料、创建学习现场、使用 AI Terminal、生成学习资源、查看知识图谱和学习画像等操作。

### 3.2 系统演示用户

系统演示用户面向答辩或评审场景，重点使用课程主页、资料库、知识图谱、AI Terminal、学习规划和 AI Studio 展示系统核心能力。

### 3.3 管理与开发人员

管理与开发人员主要负责系统部署、环境变量配置、AI 模型接入、向量检索服务启动、数据初始化、接口测试和运行状态排查。

## 4. 系统访问与基础流程

### 4.1 系统入口

前端系统通过浏览器访问，核心路由包括：

- 登录页：`/`
- 工作空间主页：`/workspaces`
- 指定课程空间：`/workspaces/:id`
- 知识图谱页面：`/workspaces/:id/knowledge-graph`
- 学习现场页面：`/workbenches/:id`

### 4.2 基础使用流程

1. 用户注册或登录系统。
2. 创建课程 Workspace，填写课程名称、描述和专业方向。
3. 上传或导入学习资料，包括文件、网页、文本、音频或视频。
4. 通过 AI Terminal 输入学习目标，生成目标草案或创建学习现场。
5. 在 Workbench 中查看资料、写笔记、编辑代码、生成学习资源并与 AI 助手互动。
6. 对课程资料进行知识索引与知识图谱构建。
7. 查看学习智能面板，分析学习状态、知识薄弱点、学习计划和后续建议。
8. 使用 AI Studio、闪卡和测验能力完成练习、复盘和巩固。

## 6. 账号与安全模块

### 6.1 注册与登录

系统提供用户注册、登录、当前用户查询和退出登录能力。用户登录后才能访问 Workspace、Workbench 和学习数据。

主要功能：

- 用户注册。
- 用户登录。
- 获取当前登录用户。
- 退出登录。
- 登录限流与注册限流。
- 会话 Cookie 与服务端 Token 管理。

具体实现中，注册流程会校验用户名、邮箱和密码强度。用户名要求为 3-24 位字母、数字或下划线，邮箱需要符合基础邮箱格式，密码由安全服务进行强度校验。注册成功后，系统会对密码进行哈希存储，并立即创建登录会话。

登录流程支持使用用户名或邮箱作为登录标识。系统会在数据库中查找对应用户，并通过密码校验后创建会话。会话信息通过 HttpOnly Cookie 写入浏览器，前端后续请求会携带该 Cookie，以保持登录状态。

前端访问核心页面时会经过 `ProtectedRoute`，后端则通过 `requireAuth` 中间件解析 Cookie、校验会话和读取当前用户。该设计使用户在刷新页面、切换 Workspace 和访问 Workbench 时无需重复登录。

代码依据：

- 前端登录页：`frontend/src/pages/LoginPage.tsx`
- 前端认证状态：`frontend/src/store/authStore.ts`
- 后端认证接口：`backend/src/routes/auth.ts`
- 注册与登录控制器：`backend/src/controllers/authController.ts`
- 会话 Cookie 中间件：`backend/src/middleware/auth.ts`
- 会话服务：`backend/src/services/authSessionService.ts`
- 密码校验与哈希：`backend/src/utils/password.ts`、`backend/src/services/authSecurityService.ts`

### 6.2 访问保护

系统前端通过受保护路由限制未登录用户访问核心页面；后端通过认证中间件、工作空间所有权校验和 Workbench 访问校验保护用户数据。

访问保护分为三个层次：

- 页面层：未登录用户只能访问登录页，访问 `/workspaces`、`/workspaces/:id`、`/workbenches/:id` 等路由时会被拦截。
- Workspace 层：后端会根据 `workspaceId` 查询 Workspace 的 `userId`，只有所有者可以读取或修改。
- Workbench 层：系统先根据 Workbench 找到所属 Workspace，再校验当前用户是否拥有该 Workspace。

该模块对比赛展示很关键，因为系统保存了学习资料、对话历史、学习画像和长期记忆，必须保证不同用户之间的数据隔离。

## 7. Workspace 课程空间模块

### 7.1 课程空间管理

Workspace 是系统的课程级容器。用户可以围绕一门课程、一个项目方向或一个长期学习主题创建 Workspace。

主要功能：

- 创建 Workspace。
- 查看 Workspace 列表。
- 查看 Workspace 详情。
- 修改课程名称、描述、专业方向和 AI Terminal 配置。
- 删除 Workspace。
- 复制 Workspace。

Workspace 的创建入口位于工作空间 Shell 页面。用户创建时可填写名称、背景描述和专业方向，其中专业方向会被后续 AI Terminal、学习规划和资源生成模块用作个性化上下文。例如同样学习数据库课程，计算机专业和经管专业的案例选择、练习侧重点和解释方式可以不同。

Workspace 列表支持侧边栏浏览、当前空间切换、搜索、菜单操作、重命名、导出、删除和复制。删除属于高风险操作，界面侧会进行确认；复制功能用于在比赛演示或课程模板复用时快速生成相似课程空间。

后端 Workspace 模型不仅保存名称和描述，还关联 Workbench、FileSystemObject、LearningGoal、LearningPlan、LearningEvent、KnowledgeChunk、CourseKnowledgeConcept、LearnerState、SavedMemory、StudioArtifact、Flashcard 等多类数据。因此，Workspace 是全系统的数据边界和学习上下文边界。

代码依据：

- 页面入口：`frontend/src/pages/WorkspaceShellPage.tsx`
- 课程详情兼容页：`frontend/src/pages/WorkspaceDetailPage.tsx`
- 前端 API：`frontend/src/api/client.ts`
- 后端路由：`backend/src/routes/workspace.ts`
- 后端控制器：`backend/src/controllers/workspaceController.ts`
- 数据模型：`backend/prisma/schema.prisma` 中的 `Workspace`

### 7.2 课程主页

课程主页用于展示课程整体状态，是用户进入学习过程的主入口。页面可聚合课程概览、最近学习活动、学习现场、资料、智能分析和 AI Terminal。

主要信息：

- 课程名称、描述和专业方向。
- 最近活动和继续学习入口。
- Workbench 列表。
- 课程资料分区。
- 学习智能看板入口。
- AI Terminal 入口。

当前 Workspace Shell 页面将左侧导航和右侧内容区整合为一个课程级操作台。左侧区域包含 Workspace 列表、当前课程下的 Workbench 列表、置顶 Workbench、搜索入口和用户退出入口；右侧区域根据标签页展示学习现场、资料、图谱、规划、画像、智能面板或 AI Terminal。

课程主页不是单纯的静态概览，而是将“继续学习”作为核心目标。系统会读取课程资料、Workbench、聊天会话、学习计划和学习状态，为用户提供继续打开 Workbench、继续对话、添加资料或生成学习计划的入口。

前端还保留了 `WorkspaceDetailPage` 作为兼容页，该页面包含课程状态、学习现场表格、资产视图和智能面板等实现。后续正式文档可根据最终演示入口，选择以 `WorkspaceShellPage` 为主或将两者关系说明为新旧布局演进。

### 7.3 Workspace 标签页

当前系统在 Workspace Shell 中组织了以下主要标签页：

- Workbenches：学习现场列表。
- Knowledge：课程资料与知识资源。
- Knowledge Graph：课程知识图谱。
- Planning：学习规划。
- Profile：学习画像。
- Intelligence：学习智能仪表盘。
- AI Terminal：课程级 AI 对话与智能执行。

各标签页的功能边界如下：

| 标签页 | 用户可见功能 | 背后关联能力 |
| --- | --- | --- |
| Workbenches | 查看、创建、打开、置顶、克隆、移动和删除学习现场 | Workbench CRUD、状态保存、资源绑定 |
| Knowledge | 上传文件、搜索网页、粘贴 URL、粘贴文本、查看课程资料卡片 | 文件系统、网页抽取、资源管理、预览和索引 |
| Knowledge Graph | 查看课程概念图谱、概念关系、标签、解释与练习 | 知识图谱构建、概念关系、知识诊断 |
| Planning | 查看计划列表、生成计划、编辑阶段、更新步骤状态 | MCL 规划、学习计划持久化、图谱约束规划 |
| Profile | 查看学习画像、长期记忆、近期事件和记忆治理 | Learner State、Saved Memory、Memory Control |
| Intelligence | 统一展示知识、诊断、规划和记忆分析 | LearningIntelligenceDashboard |
| AI Terminal | 课程级对话、资料选择、文件附件、流式输出和行动审批 | WorkspaceTerminalChat、MultiAgentOrchestrator |

## 8. Workbench 学习现场模块

### 8.1 学习现场定位

Workbench 是面向具体学习任务的工作台。例如“第 3 章 SQL JOIN 复习”“排序算法可视化学习”“Django 博客项目实践”等都可以作为一个 Workbench。

Workbench 聚合以下内容：

- 当前任务目标。
- 绑定资料。
- 笔记、代码、预览和生成结果。
- AI 助手上下文。
- 布局状态和编辑器状态。
- 与学习目标、学习计划、学习事件和学习画像相关的数据。

### 8.2 Workbench 管理

主要功能：

- 创建 Workbench。
- 查看 Workbench 列表。
- 查看 Workbench 详情。
- 修改标题、描述、布局和状态。
- 克隆 Workbench。
- 移动 Workbench。
- 删除 Workbench。
- 保存 Workbench 的编辑器状态。
- 管理 Workbench 绑定资源。
- 调整资源顺序和分组。

Workbench 的前端管理能力主要由 `workbenchApi` 和 `WorkbenchPage` 驱动。用户可以从 Workspace 创建一个新的学习现场，也可以由 AI Terminal 根据目标自动创建。创建时可传入标题、描述和初始资源 ID，系统会把选定资料绑定到 Workbench。

Workbench 支持克隆和移动。克隆适合把一个已有学习现场复制成新的练习场景，移动适合将学习现场迁移到其他课程空间。资源顺序支持拖拽调整，保存后会影响左侧资源列表和学习现场内部资料组织。

Workbench 的状态不是只保存标题和描述，还包括编辑器布局、打开的资源、激活面板、AI 助手状态、AI Studio 结果等。前端通过 `saveState` 将这些状态持久化到后端的 `stateJson`，用户下次打开时可以恢复上次的学习环境。

代码依据：

- 前端页面：`frontend/src/pages/WorkbenchPage.tsx`
- 前端 API：`frontend/src/services/workbenchApi.ts`
- 编辑器类型与布局：`frontend/src/types/workbench.ts`、`frontend/src/utils/workbenchLayout.ts`
- 后端路由：`backend/src/routes/workbenches.ts`
- 后端服务：`backend/src/services/workbenchService.ts`
- 数据模型：`backend/prisma/schema.prisma` 中的 `Workbench`、`WorkbenchResource`

### 8.3 多面板工作区

Workbench 支持多面板组织学习内容，典型面板包括：

- 资料预览面板：查看 PDF、Markdown、文本、网页来源和其他资源。
- 笔记编辑面板：记录学习笔记，支持富文本/块编辑体验。
- 代码编辑面板：查看或编辑代码类学习资料。
- AI 助手面板：基于当前学习现场进行问答、解释和总结。
- AI Studio 面板：生成测验、报告、闪卡、思维导图和可视化资源。
- 结果面板：展示 AI 生成内容或处理结果。
- Motion Canvas 可视化面板：展示算法或过程动画类学习资源。

多面板工作区的核心体验是“资料、笔记、代码和 AI 能力并排协作”。用户可从资源列表打开文件，系统会根据资源类型选择合适的编辑器或预览器。例如 Markdown 和笔记资源进入笔记编辑器，代码资源进入代码面板，PDF/文档进入资源预览面板，AI Studio 和 AI 助手则作为智能工具面板打开。

工作区支持标签页和分屏布局。用户可将资源以新标签页打开，也可以拖放到当前面板形成左右或上下分屏。编辑器激活状态、面板比例、当前文件和视图状态都会进入 Workbench 状态，用于恢复学习现场。

笔记和代码类资源支持保存内容。笔记保存可创建修订记录，后续可查看历史版本并回滚。AI 助手可读取当前激活面板、可见内容、选中文本和绑定资源，以便回答更贴近当前学习现场。

相关组件：

- 编辑器容器：`frontend/src/components/workbench/WorkbenchEditor.tsx`
- 编辑器内容：`frontend/src/components/workbench/WorkbenchEditorContent.tsx`
- 资源预览：`frontend/src/components/workbench/ResourcePanel.tsx`
- 笔记编辑：`frontend/src/components/workbench/NotesPanel.tsx`、`BlockSuiteNotesEditor.tsx`
- 代码编辑：`frontend/src/components/workbench/CodePanel.tsx`、`BlockSuiteCodeEditor.tsx`
- AI 助手：`frontend/src/components/workbench/AIAssistantPanel.tsx`

## 10. AI Terminal 模块

### 10.1 模块定位

AI Terminal 是课程级智能入口，承担类似“学习版 Codex”的角色。用户可以用自然语言描述学习目标、请求规划、询问资料内容、触发资源生成或执行需要确认的智能操作。

### 10.2 对话模式

系统支持以下模式：

- 普通 Chat：围绕当前课程资料和学习上下文进行对话。
- Agentic 模式：允许系统生成行动建议、调用能力，并在需要时请求用户批准。

普通 Chat 更适合问答、解释、总结和资料定位。Agentic 模式更适合“帮我创建一个学习现场”“根据资料生成计划”“把这些资源整理成练习”等包含系统动作的请求。Agentic 执行过程中，如果需要创建文件、生成资源、更新计划或执行其他可能影响用户数据的操作，系统会返回审批请求，用户确认后再执行。

前端的 `LearningTerminal` 负责展示消息、流式输出、来源选择、附件和审批状态；后端的学习路由会根据模式调用普通终端聊天服务或多智能体执行链。

### 10.3 会话管理

主要功能：

- 新建对话。
- 查看历史对话列表。
- 加载历史消息。
- 选择上下文资料。
- 附加文件参与对话。
- 流式响应。
- 审批需要用户确认的操作。
- 保存对话历史。

会话管理支持课程级历史对话。每个会话保存标题、模式、选中资料、附件、创建时间、更新时间、消息数量和最后消息预览。前端会按时间对历史会话进行分组，用户可继续旧会话，也可开启新会话。

用户可为对话选择资料来源，并指定 focused 或 full_context 等上下文使用模式。focused 更适合围绕某几个文件精读，full_context 更适合让 AI 综合完整资料上下文。聊天附件则适合临时加入尚未沉淀到课程资料库的文件或图片。

后端会保存对话历史，并可为历史对话补写 embedding。后续检索时，系统可以找回与当前问题相关的历史对话，但不会把历史对话直接当成长期偏好，除非用户明确保存为记忆。

- “帮我基于这门课创建一个数据库复习计划。”

## 11. AI 助手与上下文问答模块

### 11.1 上下文来源

系统的 AI 问答不是孤立聊天，而是会综合多种上下文：

- 用户当前问题。
- 当前 Workbench。
- 当前激活面板。
- 选中文本。
- 打开的资料面板。
- Workbench 绑定文件。
- Workspace 知识库检索结果。
- 最近对话。
- 显式保存记忆。
- 学习画像和学习状态。
- 聊天附件。

### 11.2 Context Capsule

系统通过 Context Capsule 将检索结果、引用、用户状态和上下文策略打包，供 AI 模型生成回答。该机制用于提高回答的资料相关性、可解释性和可追溯性。

Context Capsule 的构建过程会先收集前端传入的活跃上下文，再结合后端检索结果和学习画像。上下文策略会决定哪些层必须进入提示词、哪些层可降级、哪些内容需要裁剪。最终输出会包含引用、来源置信度、检索片段、估算 token、上下文轨迹和 fallback 原因。

系统支持的上下文模式包括：

- auto：系统自动判断使用哪些上下文。
- selection_only：仅使用用户选中文本。
- viewport：使用当前可见区域。
- active_file：优先使用当前文件。
- workbench：使用整个学习现场资源。
- model_knowledge：主要依赖模型通用知识。

### 11.3 主要能力

- 基于资料问答。
- 总结当前文件。
- 解释代码。
- 解释选中文本。
- 根据上下文生成学习建议。
- 支持流式输出。
- 保存对话记录并用于后续检索。

AI 助手支持同步和流式两种返回方式。流式接口会向前端发送 delta、timeline、meta、done 和 error 等事件，因此用户可以看到答案逐步生成，也可以看到智能体执行时间线。回答完成后，系统会返回使用的模型、runId、检索片段、上下文策略、上下文摘要和质量信息。

笔记编辑能力提供 proofread、improve、explain、reformat、shorten、expand、summarize、outline 等动作。用户在笔记中选中文本后，可让 AI 改写、解释、扩展或提纲化，结果可直接应用回笔记内容。

- 笔记编辑面板：`frontend/src/components/workbench/NotesPanel.tsx`

## 12. 知识索引与智能检索模块

### 12.1 知识索引流程

用户上传或导入资料后，系统可将资料转换为知识块并建立索引。

基本流程：

1. 提取文档文本。
2. 按章节、段落或语义块进行切分。
3. 生成知识块元数据。
4. 写入本地数据库。
5. 调用 embedding 服务生成向量。
6. 写入 Qdrant 向量库。
7. 支持关键词检索、向量检索和重排。

知识索引可针对单个文件触发，也可针对整个 Workspace 批量重建。系统提供索引任务列表和状态查询能力，便于前端展示哪些文件已经可用于检索、哪些文件正在处理中、哪些文件索引失败。

文件切块时会保留 chunk 类型、标题路径、页码、段落位置、表格/列表/代码块信息、摘要和来源哈希。这样检索命中后，AI 不只知道“文本内容”，还可以知道“来自哪个文件、哪一页、哪个结构块”。

代码依据：

- 前端 API：`frontend/src/services/learningApi.ts` 中的 `indexKnowledgeFile`、`reindexWorkspaceKnowledge`、`listKnowledgeIndexJobs`
- 后端路由：`backend/src/routes/learning.ts`
- 文档抽取：`backend/src/services/documentTextExtractionService.ts`
- 知识切块：`backend/src/services/knowledgeChunkingService.ts`
- 索引服务：`backend/src/services/knowledgeIndexingService.ts`
- 索引队列：`backend/src/services/knowledgeIndexQueueService.ts`
- Chunk Schema：`backend/src/services/chunkSchema.ts`

### 12.2 检索能力

系统检索栈包括：

- BM25/关键词召回。
- Qdrant 向量召回。
- 多路结果合并。
- MMR 去冗余。
- BGE reranker 重排。
- Context Capsule 输出。

检索入口分为知识库检索和 Workspace 文件检索。知识库检索更偏向知识块和 AI 上下文，Workspace 文件检索更偏向资源卡片与文件级命中。用户在 AI 对话、学习规划、知识图谱解释和资源生成中触发的问题，都可以复用同一套检索底座。

向量检索通过 embedding 服务生成查询向量，再到 Qdrant 中召回相似知识块；关键词检索负责补充术语、标题和精确匹配；reranker 用于对候选结果重新排序。系统还会对结果做合并、去冗余和数量控制，避免提示词被重复片段占满。

- 检索服务：`backend/src/services/knowledgeSearchService.ts`
- 向量库服务：`backend/src/services/vectorStoreService.ts`

- Workspace 文件索引：`backend/src/services/workspaceFileIndexService.ts`
- 检索配置：`docker-compose.context.yml`、`backend/.env.example`

## 13. 课程知识图谱模块

核心对象包括：

- 知识概念。
- 概念标签。

- 支持图谱重建和质量增强。

课程知识图谱构建可以同步执行，也可以启动后台构建任务。后台任务适合较大的 Workspace，因为知识索引、资料读取、AI 抽取、概念规范化、关系生成和质量校验可能需要较长时间。前端会记录构建任务 ID，并轮询任务状态。

图谱构建不仅从文件中抽取概念，还会处理概念清洗、别名、难度、证据、标签、关系权重、置信度和来源。对于 AI 抽取结果，系统提供验证、治理、清理污染概念和增强质量等接口，以减少噪声概念进入正式图谱。

### 13.3 图谱分析

主要能力：

- 查看课程知识图谱。
- 查看概念详情与标签。
- 获取概念解释。
- 查看概念相关练习。
- 分析先修缺口。
- 查看薄弱知识邻域。

前端知识图谱支持图视图、列表视图和路径视图。图视图可展示节点、边、关系类型、薄弱/掌握/准备就绪等分组，并支持缩放、拖拽、布局记忆、关系过滤、节点标签显示和局部图谱查看。用户点击概念后，可查看入边、出边、来源、标签和相关练习。

分析接口面向学习补弱。先修缺口用于找出当前目标概念之前还未掌握的概念；薄弱邻域用于查看薄弱概念周围的相关知识；补救路径用于按先修关系安排复习顺序；准备度分析用于判断用户是否适合进入某个知识点。

规划界面包含计划列表、步骤视图和反馈视图。计划列表展示已有计划，用户可以新建计划、打开计划、应用到当前 Workbench、创建新 Workbench 或删除计划。步骤视图展示结构化阶段，用户可更新阶段状态，也可请求 AI 解释某个阶段或提出修改建议。反馈视图用于聚合计划、测验、练习和复盘相关事件。

学习计划支持版本化。用户对计划提出反馈、审查或回滚时，系统可以生成新版本并保留 previousPlanId。这样既能支持动态调整，又能在测试说明书中展示“计划迭代与治理”的证据。

### 14.3 多智能体规划

系统内部包含学习状态构建、规划、资源推荐、生成、审查和持久化等步骤。相关过程会记录为 LearningRun 和 RunStep，用于展示智能体执行轨迹和后续评估。

多智能体规划不是一次性生成文本，而是先构建学习状态，再综合目标、资料、知识图谱、用户画像和约束条件生成计划。运行过程会被记录为 run 和 step，前端可查询运行状态，后端可根据 runId 追踪失败原因、耗时和阶段输出。

## 15. 学习画像与记忆模块

- 关注概念。
- 候选薄弱点。
- 稳定薄弱点。

### 15.2 显式记忆

系统支持 Saved Memory，即用户明确要求长期记住的信息。例如学习偏好、专业方向、长期目标或特殊约束。

主要功能：

- 删除记忆。
- 查看记忆解释。
- 管理候选记忆。

用户可新增、修改、删除和恢复记忆。系统同时支持候选记忆机制：AI 可以从对话中发现可能值得保存的信息，但需要用户确认保存或忽略，以避免无意中把临时表达写成长期记忆。

## 17. AI Studio 资源生成模块

### 17.1 模块定位

AI Studio 用于把课程资料、学习目标和上下文转换为学习资源。它既可以服务于单个 Workbench，也可以服务于整个 Workspace。

### 17.2 资源类型

系统支持或预留的资源类型包括：

- 学习报告。
- 幻灯片。
- 思维导图。
- 闪卡。
- 测验。
- 数据表。
- 结构化可视化。
- 过程动画。
- 练习资源。

AI Studio 以“学习目标类别 + 资源模板”的方式组织功能。当前模板类别包括 Understand、Map、Practice、Review、Lab、Visualize、Plan。每个模板会指定 generator、renderer、输出格式、文件名、使用建议和标签。前端会优先读取后端模板，也保留核心模板作为 fallback。

典型模板包括：

- Resource Understand：把资料转成可编辑笔记。
- Resource Compare：比较多个资料的差异和互补关系。
- Custom Practice：根据目标生成测验。
- Flashcards：生成复习卡片。
- Slide Deck：生成课件或汇报幻灯片。
- Mind Map：生成思维导图或概念图。
- Interactive Demo：生成交互式演示资源。
- Study Plan：生成 Workbench 级战术学习计划。

### 17.3 生成流程

AI Studio 后端采用模板化和工作流式设计。

典型步骤：

1. ContextAgent 收集上下文。
2. LearnerStateAgent 读取学习状态。
3. ResourcePlanningAgent 规划资源结构。
4. GeneratorAgent 生成资源内容。
5. ReviewAgent 进行质量检查。
6. PublisherAgent 发布为系统资源。

用户在 AI Studio 中可以选择目标类别和模板，填写生成要求，选择资料来源，然后点击生成。对于 Resource Notes 和 Resource Compare 类型模板，系统强调“只使用用户勾选的来源”，避免把当前打开文件或其他上下文误混入生成结果。生成完成后，结果会显示在 AI Studio 面板中，同时保存为系统资源或 Studio Artifact。

生成结果根据类型进入不同查看器：测验会进入 Quiz Viewer，闪卡进入 FlashcardDeckViewer，幻灯片可展示 PPTX 生成信息，思维导图可切换 2D/3D 视图并导出 JSON/PNG，数据表按结构化内容展示。

代码依据：

- 前端面板：`frontend/src/components/workbench/AIStudioPanel.tsx`
- 前端 API：`frontend/src/services/aiApi.ts`
- 后端路由：`backend/src/routes/studio.ts`
- Studio 说明：`backend/src/services/studio/README.md`
- 模板注册：`backend/src/services/studio/templateRegistry.ts`
- 工作流：`backend/src/services/studio/workflow.ts`
- 生成器：`backend/src/services/studio/generators.ts`
- Artifact 渲染：`backend/src/services/studio/artifactRenderer.ts`
- V2 服务：`backend/src/services/studio/studioV2Service.ts`

### 17.4 质量控制

系统对生成物进行结构化约束和审查，特别是对测验、可视化资源和教学过程动画，会检查格式、来源、交互性和内容覆盖情况。

AI Studio 会返回 review、qualityReport、workflowTrace、recommendation、structured、artifact、renderJob、delivery 等元数据。测验资源会进行题目质量检查，视觉化资源会通过 Teaching Visualization IR 约束，检查教学理解、过程轨迹、视觉映射和交付格式是否完整。渲染型资源还会创建 render job，支持查看任务状态和失败重试。

### 22.1 创建课程空间并导入资料

1. 登录系统。
2. 进入 Workspace 列表。
3. 创建新的课程空间。
4. 填写课程名称、背景描述和专业方向。
5. 上传 PDF、Markdown、代码、音视频或其他资料。
6. 导入网页来源或粘贴文本来源。
7. 查看资料是否出现在课程资料库中。

### 22.2 通过 AI Terminal 创建学习现场

1. 进入某个 Workspace。
2. 打开 AI Terminal。
3. 输入学习目标。
4. 系统生成目标草案，包括标题、目标、技能和薄弱点。
5. 用户确认后创建 Workbench。
6. 系统生成初始资源，如目标蓝图、诊断练习、学习计划和笔记模板。

### 22.3 在 Workbench 中学习

1. 打开 Workbench。
2. 从资源面板选择学习资料。
3. 在资料预览面板查看内容。
4. 在笔记面板记录理解和问题。
5. 在 AI 助手面板提问或请求解释。
6. 使用 AI Studio 生成练习、闪卡或总结。
7. 保存学习结果并回到课程空间查看进度。

### 22.4 构建知识图谱并进行补弱

1. 对课程资料进行知识索引。
2. 启动课程知识图谱构建。
3. 查看图谱中的概念、关系和先修依赖。
4. 查看薄弱知识邻域。
5. 获取补救路径和准备度分析。
6. 根据建议创建新的学习计划或 Workbench。

### 22.5 使用闪卡复习

1. 从 AI Studio 或资料生成闪卡。
2. 进入闪卡复习入口。
3. 查看到期卡片。
4. 完成复习并提交掌握情况。
5. 对不理解的卡片请求 AI 解释。
